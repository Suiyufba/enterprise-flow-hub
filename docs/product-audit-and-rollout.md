# Enterprise Flow Hub 产品审计与落地方案

## 产品定位

Enterprise Flow Hub 不是通用聊天机器人，而是中小企业的业务操作 Agent。它应围绕客户、订单、付款、发票、资料、待办和自动化完成“查询证据、执行动作、记录结果、人工复核”的闭环。

## 本轮审计结论

| 维度 | 改造前主要问题 | 本轮落地 |
| --- | --- | --- |
| Agent | 工具可能由模型模拟，无法证明执行成功 | Claude Code 内核挂载进程内 MCP，工具结果落库，错误不再标成功 |
| 权限 | 前端公开 Key 与可伪造用户头并存 | 产品 API 统一使用 96 位会话令牌；未登录只可访问登录和健康检查 |
| 数据 | Agent 不知道发票/订单的真实入口 | 企业范围的只读业务 MCP，固定资源白名单，不接受任意 SQL |
| 写操作 | 客户、状态修改和去重缺少受控入口 | 业务操作 MCP 使用操作白名单、企业范围和状态校验 |
| 自动化 | 多种动作只更新运行次数，没有真实执行 | 只开放 AI、通知、业务工具三类动作；每次记录输出、错误和耗时 |
| 触发器 | 定时只识别每天，消息/文件未连通，Webhook 无鉴权 | 支持每日、工作日、分钟/小时间隔；消息、项目文件、手动和带密钥 Webhook 已接通 |
| 通知 | 飞书/企微参数不一致，也无法试发 | 两个平台分别生成正确 payload；插件页保存后可发送测试消息 |
| 规则 | 对象和动作枚举不一致，事件没有产生 | CRM、订单、发票、付款、文件和项目事件接入规则；动作失败不伪装成功 |
| 落地 | 规则/Agent 创建待办后没有承接页面 | 新增待办中心，可查看、开始和完成任务 |
| UI | 登录页色彩脱离主系统，运行错误不醒目，移动端侧栏难以浏览 | 登录页统一视觉；侧栏改为一致线性图标和可滚动移动抽屉；自动化卡显示最近状态、错误、耗时和 Webhook 调用方式 |

## MCP 与 Skill 边界

MCP 是可验证的最小能力，负责读写系统；Skill 是业务方法，负责决定何时组合 MCP。

| MCP | 风险 | 能力边界 |
| --- | --- | --- |
| 项目上下文 MCP | 只读 | 只读取当前对话的企业和项目资料 |
| 业务数据查询 MCP | 只读 | 查询 dashboard、客户、订单、付款、发票、待办、自动化、资料库 |
| 业务操作 MCP | 写入 | 创建客户/待办，更新状态，按电话合并重复客户 |
| 项目表格分析 MCP | 只读 | 按当前企业/项目读取已上传的 CSV、TSV、TXT、XLSX、XLSM，返回精确行数、表头、样例和空值统计 |
| 网页可用性巡检 | 只读 | 检查公开页面；阻断本机、私网、DNS 私网解析和私网重定向 |
| 通知 MCP | 外部写入 | 仅向已绑定并启用的飞书/企微 Webhook 发送文本 |
| 自动化创建工具 | 写入 | 只创建 AI、通知或业务工具动作，不开放 Shell 和任意 API |

已落地 Skill：发票异常处置、客户跟进运营、订单回款核对、经营日报、业务数据治理。默认角色按职责加载 Skill，未被 Skill 授权的写工具不会暴露给 Agent。

## 真实触发能力

| 触发方式 | 状态 | 入口 |
| --- | --- | --- |
| 定时 | 可用 | 每日/工作日指定时间、每 N 分钟、每 N 小时 |
| 消息 | 可用 | 当前项目产生用户消息后触发 |
| Webhook | 可用 | `POST /api/automations/:id/webhook`，请求头携带自动化密钥 |
| 文件 | 可用 | 文件管理选择项目并上传后触发 |
| 手动 | 可用 | 自动化卡片“测试运行” |
| 邮件 | 未接入 | 需要邮箱 OAuth/IMAP 凭据和独立连接器；编辑器禁止新建，避免假触发 |

## 上线验收标准

1. 匿名访问业务 API 返回 401，登录后 Agent 状态和工作区正常加载。
2. Agent 查询发票时出现真实 MCP 调用，回答数字与数据库一致。
3. Agent 创建待办后能在待办中心查看并完成。
4. 自动化测试运行产生一条运行记录；失败时显示错误，运行次数也记录尝试。
5. 文件上传必须选择项目，上传后对应文件触发器产生运行记录。
6. Webhook 缺少或使用错误密钥时返回 401，正确密钥才执行。
7. 飞书/企微只有在绑定、启用且平台返回成功码后才显示发送成功。
8. 新库初始化、现有库迁移、TypeScript 构建、单元测试和桌面/移动浏览器回归全部通过。
9. 工作流编辑器只允许保存已经接通的动作，旧 Shell/API/浏览器动作不能重新启用。
10. 生产数据迁移前后业务表数量一致，`integrity_check` 为 `ok` 且外键检查为空。

## 外部依赖

飞书和企业微信的真实群消息需要用户在插件页填写各自群机器人的 Webhook。没有外部凭据时可以验证格式、失败路径和绑定流程，但不能替用户生成有效机器人地址。邮件触发保留为后续连接器，不计入当前可用能力。
